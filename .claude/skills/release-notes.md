# 릴리스 노트 작성 스킬

## 개요
버전별 변경사항을 정리하는 릴리스 노트 작성 스킬입니다.

## 릴리스 노트 구조

```markdown
# v[버전] 릴리스 노트

**릴리스 날짜:** YYYY-MM-DD

## 주요 변경사항
- 핵심 변경 1~3개 요약

## 신규 기능
- [기능명]: 설명

## 개선사항
- [영역]: 개선 내용

## 버그 수정
- [이슈번호] 버그 설명 및 수정 내용

## 알려진 이슈
- [이슈번호] 이슈 설명

## 업그레이드 가이드
이전 버전에서 업그레이드 시 필요한 작업
```

## 저장 위치
`docs/release-notes/v[버전].md`

## 버전 규칙
- Major.Minor.Patch (예: 0.1.30)
- Patch: 버그 수정, 작은 개선
- Minor: 새 기능 추가
- Major: 큰 변경, 호환성 변경

## 작성 지침
1. git log에서 변경사항 수집
2. 사용자 관점으로 변경사항 재작성
3. 기술 용어는 이해하기 쉽게 설명
4. 영향 범위 명시
